# Google Analytics 4 基礎（GA4 for SEO）

## 核心概念
Google Analytics 4（GA4）是 Google 的免費網站分析工具，記錄用戶在你網站上的所有行為。對 SEO 而言，GA4 告訴你流量**來自哪裡**、用戶**做了什麼**、有沒有**達成目標**。

**GA4 vs 舊版（UA/GA3）：**
- 2023 年 7 月，Google 已停止舊版 Universal Analytics 的數據收集
- 如果你的網站還沒設定 GA4，立刻設定——越晚設定，歷史數據越少

---

## GA4 基本設定

### 如何安裝 GA4

**方法 1：直接在網站加入程式碼（靜態 HTML）**
```html
<!-- 加在 <head> 裡 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

**方法 2：透過 Google Tag Manager（GTM）安裝（推薦）**
- 安裝 GTM → 在 GTM 裡加 GA4 代碼
- 優點：之後加其他追蹤代碼不需要改網站程式碼

**方法 3：CMS 外掛（WordPress）**
- Site Kit by Google（官方外掛，免費）
- MonsterInsights

### 確認 GA4 是否正確安裝
- 進入 GA4 → 管理 → 資料串流 → 選你的串流 → 查看即時報表
- 同時打開你的網站，如果 GA4 即時頁面顯示有用戶，就安裝成功

## SEO 最常用的 GA4 報表

### 1. 自然搜尋流量（最重要）

**路徑：** 報表 → 獲取 → 流量獲取

**看什麼：**
- 「工作階段預設管道群組」欄位找「自然搜尋（Organic Search）」
- 點擊次數（工作階段）、使用者數、互動率、平均互動時間

**進階：** 點擊「自然搜尋」這一行 → 次要維度選「到達網頁」→ 看哪些頁面帶來最多自然流量

---

### 2. 到達頁面分析

**路徑：** 報表 → 參與 → 到達網頁和畫面

**篩選自然流量：** 點擊「新增篩選器」→ 工作階段預設管道群組 = 自然搜尋

**看什麼：**
- 哪些頁面帶來最多自然流量
- 各頁面的「互動工作階段」和「平均互動時間」（停留時間短 = 可能內容不符合用戶期待）
- 「互動率」（低於 40% 可能要注意）

---

### 3. 用戶行為流（理解用戶路徑）

**路徑：** 探索 → 建立新探索 → 自由格式

用來分析：
- 用戶從自然搜尋進來後，接下來去了哪裡
- 有多少比例的用戶完成了你的目標（購買、填表、訂閱）

---

### 4. 轉換事件（Conversions）

**是什麼：** 你定義的「成功行為」，例如：購買完成、表單送出、電話點擊

**GA4 預設追蹤：** first_visit、session_start、page_view

**自訂轉換（需要設定）：**
- GA4 → 管理 → 事件 → 建立事件
- 或透過 GTM 設定特定按鈕點擊追蹤

**為什麼對 SEO 重要：**
知道自然搜尋流量有多少比例最終轉換，才能真正評估 SEO 的商業價值

## GA4 + GSC 整合（強力組合）

### 連結步驟
GA4 → 管理 → 產品連結 → Search Console 連結 → 選取你的 GSC 資源

### 連結後能看什麼
**路徑：** 報表 → 獲取 → Search Console

- **Google 自然搜尋查詢**：每個關鍵字帶來多少點擊（來自 GSC）
- **Google 自然搜尋流量**：結合了 GSC 的排名數據和 GA4 的行為數據

這是最接近「關鍵字 → 用戶行為 → 轉換」完整路徑的免費工具組合。

## 常見的 GA4 問題

### 數據差異：GA4 和 GSC 的數字不同
這是正常的，因為：
- GSC 記錄的是點擊次數（用戶點了搜尋結果）
- GA4 記錄的是工作階段（用戶成功載入頁面）
- 使用者封鎖追蹤腳本（Ad Blocker）→ GA4 看不到，GSC 看得到
- 不一致比例通常在 10-20%，超過 30% 才需要排查

### 自然流量為零
可能原因：
- GA4 安裝錯誤（代碼沒有正確放置）
- 設定了 IP 過濾排除了自己的瀏覽
- 查看的日期範圍太短

### 如何確認安裝正確
- Chrome 安裝「Google Tag Assistant」擴充功能
- 打開你的網站，Tag Assistant 會顯示偵測到的 Google 標籤

## GA4 的基礎指標解讀

| 指標 | 意義 | 健康數值參考 |
|------|------|------------|
| 使用者（Users） | 獨立訪客數 | 視規模而定 |
| 工作階段（Sessions） | 訪問次數（一人可有多次） | 通常 > 使用者數 |
| 互動率（Engagement Rate） | 有互動的工作階段比例 | > 50% 較健康 |
| 平均互動時間 | 用戶平均停留時間 | > 1 分鐘較好 |
| 跳出率（Bounce Rate） | GA4 的跳出率 = 未互動工作階段比例 | < 50% 較健康 |
| 事件次數 | 各類用戶行為的觸發次數 | 追蹤目標行為 |

**注意：** GA4 的「跳出率」定義和舊版 GA 不同——GA4 跳出 = 沒有任何互動行為就離開，不只是只看一頁。
