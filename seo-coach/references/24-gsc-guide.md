# Google Search Console 完整操作指南

## 是什麼
Google Search Console（GSC）是 Google 提供的免費工具，讓你看到 Google 怎麼看你的網站——爬行狀況、索引狀態、搜尋表現、技術問題。

**類比：** 如果你的網站是一間店，GSC 就是 Google 給你的「店面健檢報告 + 顧客流量統計」。

---

## 第一次設定 GSC（從零開始）

### Step 1：前往 GSC
`search.google.com/search-console` → 用 Google 帳號登入

### Step 2：新增資源（你的網站）
點擊左上角「新增資源」，選擇：
- **網域**（推薦）：輸入 `yourdomain.com`，涵蓋所有子網域和 http/https 版本
- **網址前置字元**：輸入完整 URL，只追蹤該特定版本

### Step 3：驗證網站所有權
Google 需要確認你是網站的擁有者，有幾種方式：

| 驗證方式 | 適合對象 | 難度 |
|---------|---------|------|
| HTML 檔案上傳 | 可以存取伺服器的人 | 簡單 |
| HTML meta tag | 能編輯 `<head>` 的人 | 簡單 |
| Google Analytics（已安裝 GA4） | 已裝 GA4 的網站 | 最簡單 |
| Google Tag Manager（已安裝 GTM） | 已裝 GTM 的網站 | 最簡單 |
| DNS 記錄 | 能管理網域 DNS 的人 | 稍複雜 |

**WordPress 最簡單的方式：**
Yoast SEO → 一般 → Webmaster Tools → 貼上 Google 驗證碼

**驗證後等 24-48 小時**，GSC 才開始收集數據。

### Step 4：提交 Sitemap
左側選單 → Sitemap → 輸入你的 sitemap URL → 提交

常見 sitemap 網址：
- WordPress：`yourdomain.com/sitemap_index.xml` 或 `yourdomain.com/sitemap.xml`
- Shopify：`yourdomain.com/sitemap.xml`
- 靜態網站：自行確認位置

## GSC 的核心功能區域

### 1. 成效（Performance）— 最常用

**路徑：** 左側 → 成效 → 搜尋結果

**四個核心指標：**
| 指標 | 意義 |
|------|------|
| 點擊次數 | 用戶點擊你的搜尋結果的次數 |
| 曝光次數 | 你的頁面出現在搜尋結果的次數 |
| 點擊率（CTR） | 點擊 ÷ 曝光，代表結果的吸引力 |
| 平均排名 | 你出現的平均位置（1 = 最頂部） |

**四個維度分析（底部分頁）：**
- **查詢**：用戶搜尋什麼字找到你的 → 最重要的 SEO 情報
- **網頁**：哪些頁面有流量
- **國家**：流量來自哪裡
- **裝置**：手機 vs 電腦的比例

**實用技巧 — 找「低垂的果實」：**
篩選「平均排名 > 10」且「曝光次數 > 100」的查詢
→ 這些字你已在第二頁，稍微優化就可能上第一頁

**比較日期範圍：**
右上角「比較」→ 選「與前一期間比較」→ 看哪些字排名上升/下降

---

### 2. URL 檢查工具

**路徑：** 頂部搜尋欄貼入任何你的頁面 URL

**能做什麼：**
- 查看 Google 最後一次爬取這個頁面是什麼時候
- 確認頁面是否已被索引
- 查看 Google 實際看到的頁面內容（「查看已爬取的網頁」）
- 手動請求索引（發布新內容後使用）

**手動請求索引的時機：**
- 發布新文章後
- 修改了重要頁面的內容
- 修復了技術問題後想讓 Google 快快更新

---

### 3. 索引（Indexing）

#### 頁面（Pages）
**路徑：** 左側 → 索引 → 頁面

**看什麼：**
- **已索引的頁面數** vs 你實際的頁面數（差距大 = 有問題）
- **未索引的原因**（點擊各原因可看到受影響的 URL）

**常見的未索引原因和處理方式：**
| 原因 | 意義 | 處理 |
|------|------|------|
| 已爬取但目前未收錄 | Google 看過但不確定要不要收 | 提升頁面品質 |
| noindex 標籤 | 你告訴 Google 不要收 | 確認是否故意設定 |
| 重複的頁面，Google 選了非標準網址 | Canonical 問題 | 檢查 canonical 設定 |
| 找到但目前未編入索引 | Google 已發現但還沒爬 | 提交 URL 或等待 |
| 404 找不到 | 頁面不存在 | 修復或重定向 |

#### Sitemap
**路徑：** 左側 → 索引 → Sitemap

查看你提交的 sitemap 狀態，有沒有錯誤。

---

### 4. 體驗（Experience）

#### Core Web Vitals
**路徑：** 左側 → 體驗 → Core Web Vitals

- 分「手機」和「電腦」兩個版本
- 顯示「良好」「需要改善」「不良」的 URL 數量
- 點進去可以看哪些 URL 有問題，以及具體是哪個指標不達標

#### 手機體驗檢查
Search Console 的「行動裝置可用性」報告與 Mobile-Friendly Test 已退役。手機體驗仍然重要，但改用：
- PageSpeed Insights / Lighthouse
- Chrome DevTools 裝置模擬
- 實機檢查（文字是否可讀、按鈕是否好點、內容是否超出螢幕）

---

### 5. 搜尋外觀（Search Appearance）

**路徑：** 左側 → 搜尋外觀

查看你的網站哪些頁面出現了 Rich Results（豐富搜尋結果），以及有沒有錯誤。

---

### 6. 連結（Links）

**路徑：** 左側 → 連結

- **外部連結**：哪些網站連結到你，哪些頁面被連最多次
- **內部連結**：你網站內部哪些頁面被連接最多次（= 你認為最重要的頁面）

---

## GSC 的重要限制

- **數據延遲**：GSC 的數據通常延遲 2-3 天，不是即時的
- **歷史數據**：只保留 16 個月的成效數據（超過就看不到了）
- **排名是平均值**：「平均排名」是加權平均，不代表任何人實際搜尋看到的位置
- **曝光次數**：用戶滾動到你的結果才算曝光，不是在頁面上出現就算

---

## GSC 每月必看清單

```
每月例行 GSC 檢查（15 分鐘）

□ 成效 → 點擊數趨勢（vs 上月）
□ 成效 → 查詢 → 排名 11-20 的字（機會）
□ 成效 → 查詢 → 排名下降的字（警報）
□ 索引 → 頁面 → 新增的未索引錯誤
□ Core Web Vitals → 有沒有新的「不良」頁面
□ 手機實機 / Lighthouse → 有沒有明顯可用性問題
□ Sitemap → 確認狀態正常
```
