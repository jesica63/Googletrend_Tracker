# 31 — 初學者友善來源地圖

用途：當用戶是 SEO 新手、第一次接觸某個概念、容易被術語嚇到，或明顯需要更白話的教學時，先讀這份。

原則：**Google 當底線、Aleyda 排順序、Ahrefs 講白話、Lily 補信任、Marie 管預期、Whitespark 補 Local SEO。**

不要一次把所有人的觀點都倒給用戶。先看用戶卡在哪個層級，再選一個最適合的角度補充。

---

## 來源使用順序

### 1. Google Search Central — 當成底線

適合用途：
- 解釋 SEO 最基本的定義
- 解釋 crawl / index / title / meta / sitemap / links 的基本觀念
- 矯正常見迷思
- 調整用戶對 SEO 時程的期待

初學者版本要點：
- SEO 沒有神祕技巧，不是「做了就第一名」
- Google 會自動探索大多數網站，但你要讓它更容易理解內容
- 改完 SEO 不會立刻生效，通常要等幾週再看
- 先確認 Google 看不看得到、讀不讀得懂，再談排名

教學時可以借用的框架：
- 先問：「Google 看得到這一頁嗎？」
- 再問：「Google 知道這頁在講什麼嗎？」
- 最後才問：「這頁值不值得被排前面？」

適合補到的模組：
- `01-crawlability.md`
- `02-indexing.md`
- `03-technical-seo.md`
- `04-on-page.md`
- `19-seo-myths.md`
- `24-gsc-guide.md`

來源：
- [Google SEO Starter Guide](https://developers.google.com/search/docs/fundamentals/seo-starter-guide)
- [Google Developer's Guide to Search](https://developers.google.com/search/docs/fundamentals/get-started-developers)
- [Google SEO Mythbusting series announcement](https://developers.google.com/search/blog/2019/06/a-new-series-on-seo-for-web-developers)

### 2. Aleyda Solis / LearningSEO.io — 當成學習順序

適合用途：
- 幫新手排先後順序
- 幫 skill 保持「先 fundamentals、再 process、再 specialized」
- 避免一開始就把進階議題丟給用戶

初學者版本要點：
- 先學 fundamentals：keyword、technical、content、links
- 再進入 process：audit、measurement、reporting
- 再看 CMS、specialization、automation

教學時可以借用的框架：
- 先把當前問題放回 roadmap 裡
- 告訴用戶「你現在做到哪一段」
- 讓新手知道自己不是什麼都不會，只是正在第一層

適合補到的模組：
- `00-session-flow.md`
- `09-keyword-basics.md`
- `21-seo-measurement.md`
- `24-gsc-guide.md`

### 3. Ahrefs — 當成白話解釋器

適合用途：
- 把關鍵字、搜尋意圖、on-page、連結這些觀念講得更容易懂
- 提供新手容易理解的框架和例子
- 幫 skill 補上「先做什麼、為什麼」的白話說法

初學者版本要點：
- SEO 的基本功不複雜，重點是持續做對
- 關鍵字不要只看搜尋量，要看流量潛力和商業價值
- 搜尋意圖比關鍵字塞幾次更重要
- Google 通常會自己找到網站，但 sitemap 可以幫忙加速整理

教學時可以借用的框架：
- 「這個主題有人搜嗎？」
- 「排上去之後有沒有實際價值？」
- 「你這頁有沒有對準搜尋者真正想看的東西？」

適合補到的模組：
- `04-on-page.md`
- `05-content-quality.md`
- `09-keyword-basics.md`
- `10-links-backlinks.md`
- `21-seo-measurement.md`

來源：

### 4. Lily Ray — 當成 E-E-A-T 白話補充

適合用途：
- 解釋 E-E-A-T 到底是什麼
- 解釋為什麼有些產業更需要信任感
- 提醒用戶不要把 E-E-A-T 當成神秘分數

初學者版本要點：
- E-E-A-T 不是一個可以填表拿分的 SEO 技巧
- 它關乎「這個內容、作者、網站值不值得信任」
- YMYL 題材更需要這些訊號
- Trust 是最不能忽略的部分

教學時可以借用的框架：
- 這頁是誰寫的？
- 為什麼讀者應該相信你？
- 這個網站有沒有把真實性和可信度講清楚？

適合補到的模組：
- `05-content-quality.md`
- `06-eeat.md`
- `12-scenarios.md`

來源：
- [Lily Ray on E-E-A-T](https://lilyray.nyc/e-a-t-expertise-authoritativeness-trustworthiness/)

### 5. Marie Haynes — 當成更新後的預期管理

適合用途：
- 解釋 core update 不是處罰
- 提醒用戶流量掉了不要立刻亂改一堆東西
- 幫 skill 保持「先判斷，再行動」的節奏

初學者版本要點：
- Core update 不是收到處罰單
- 如果流量掉，是 Google 覺得別人的內容更有幫助
- 重點不是找捷徑，而是提升品質、信任和幫助性
- 真正的恢復通常也需要等後續更新或更長時間觀察

教學時可以借用的框架：
- 先確認是不是剛好卡在更新時點
- 再確認掉的是哪些頁、哪些意圖
- 最後才談該補什麼，不要先恐慌

適合補到的模組：
- `12-scenarios.md`
- `19-seo-myths.md`
- `21-seo-measurement.md`

來源：
- [Marie Haynes on Google Core Updates](https://www.mariehaynes.com/google-core-updates/)

### 6. Whitespark — 當成 Local SEO 初學者框架

適合用途：
- 教本地商家理解服務頁、首頁內容、標題、內鏈、速度、NAP
- 把 Local SEO 拉回實際網站基礎，而不是只講地圖排名

初學者版本要點：
- 實體商家不能只靠首頁吃全部關鍵字
- 每個主要服務值得有自己的一頁
- Local SEO 也離不開 title、內容、內鏈、速度、行動版
- 不要迷信 meta keywords 或舊式過度優化

教學時可以借用的框架：
- 你的每個主要服務，網站上有沒有一個清楚頁面？
- 首頁是品牌頁，還是硬塞全部服務？
- 本地商家的網站基礎有沒有跟 GBP 互相支撐？

適合補到的模組：
- `04-on-page.md`
- `07-internal-links.md`
- `08-page-speed.md`
- `22-local-seo.md`

來源：
- [Whitespark Website Optimization Basics For Local SEO](https://whitespark.ca/blog/website-optimization-basics-for-local-seo/)

## Skill 內部使用規則

- **Google 優先**：只要和 Google 官方說法衝突，以 Google 為準
- **Aleyda 排順序**：用來決定先教什麼，不用來增加深度
- **Ahrefs 白話化**：用於解釋，不用於擴大邊界
- **Lily / Marie**：只在 E-E-A-T、核心更新、品質判讀這些地方補充，不要因此把 skill 帶進顧問層
- **Whitespark**：只在 Local SEO 用戶出現時讀，不要預設載入

---

## 初學者教學話術方向

- 少用「演算法」「權重」這種抽象詞，先用「Google 看不看得懂」來講
- 少用一次講完整地圖，先講「你現在最該先看哪個點」
- 少講密技，多講基本功
- 少講保證，多講驗證
- 少講一次做完，多講先做一個可驗證的小步驟

這份 source map 是幫 skill **變得更好教**，不是讓 skill **變得更深顧問**。
